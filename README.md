# Morning Briefing

A personal daily intelligence briefing, generated by Claude and delivered to a private webpage every morning. Runs automatically on GitHub Actions — no server, no laptop required.

**[Set up yours →](./setup.html)**

---

## What you get

Six sections, generated fresh each weekday:

- **Top Stories** — 3–5 prioritised stories with full analysis, clickable source links, and what to watch next
- **Sector Scan** — all your topic areas at a glance; green dot means real news today
- **Week Ahead** — scheduled events, earnings, policy decisions, and launches for the rest of the week
- **Pattern Watch** — speculative cross-sector analysis connecting dots across your topics
- **Deep Dive** — one long-form piece each day on a timely wildcard subject; rotates automatically, never repeats
- **Signal Scan** — recent output from a curated list of thinkers you track, pulled from RSS feeds and X; refreshes weekly by default

Everything is accessible via a tab bar. Stories and pattern cards expand on click. An archive of past briefings is one tap away.

## Quick start

**You'll need:** a [GitHub account](https://github.com) (free) and an [Anthropic API key](https://console.anthropic.com) (~$3–4/month at weekday-only cadence).

### 1. Create your repo from this template

Click **Use this template → Create a new repository** at the top of this page. Make it public (required for free GitHub Pages hosting).

### 2. Configure your briefing

Open **[setup.html](./setup.html)** on this repo's GitHub Pages site. It lets you:
- Edit your topic areas (add, remove, reorder by priority)
- Choose your preferred sources
- Toggle sections on or off
- Set your schedule and timezone
- Add the thinkers you want to track in Signal Scan

Click **Save to GitHub** — it commits `config.yaml` directly to your repo. You'll need a [fine-grained GitHub token](https://github.com/settings/tokens?type=beta) with **Contents: Read and write** on your repo.

### 3. Add your Anthropic API key

In your repo: **Settings → Secrets and variables → Actions → New repository secret**

- Name: `ANTHROPIC_API_KEY`
- Value: your key from [console.anthropic.com](https://console.anthropic.com)

### 4. Enable GitHub Pages

**Settings → Pages → Source → GitHub Actions**

### 5. Run it

**Actions → Morning Briefing → Run workflow**

Your briefing will be live at `https://yourusername.github.io/your-repo-name/` within a few minutes. After that it runs automatically on the schedule in your config.

## Configuration

Everything lives in `config.yaml`. Use the [setup page](./setup.html) to edit it visually, or open it directly — every field is commented.

| Field | What it controls |
|---|---|
| `topics` | Subject areas in priority order. The briefing weights earlier topics more heavily. |
| `sources.preferred` | Cited first whenever available. |
| `sources.supplementary` | Used only when preferred sources don't cover a topic. |
| `sections` | Toggle any section on/off. Disabled sections are skipped entirely, saving API cost. |
| `signal_scan_frequency` | `"weekly"` (default) caches Signal Scan for 7 days. `"daily"` runs it every time. |
| `voices` | The thinkers tracked in Signal Scan. Each entry takes a name, RSS feed URL, and X handle. |
| `schedule` | Two cron entries (for standard and daylight saving time). The setup page converts your local time to UTC automatically. |

## Signal Scan

The Signal Scan section tracks a configurable list of thinkers — checking their RSS feeds and X posts for recent output, then summarising what each person is writing or saying and flagging anything that connects to today's top stories.

Each voice gets a card with a signal indicator: green (substantive new piece), amber (light post), grey (nothing this week).

**By default it runs weekly** and caches the results. This keeps the daily runtime fast (~3 min) and cost low. To force a fresh scan at any time, trigger the workflow manually with the **Force regenerate** option ticked.

To add or remove voices, edit the `voices` list in `config.yaml` via the setup page or directly on GitHub.

## Performance

The script runs four API calls per briefing, structured in two parallel stages:

**Stage 1 (parallel):** main briefing + deep dive + RSS fetch — the slowest of these determines stage 1 time (~3–4 min).

**Stage 2 (parallel):** signal scan (if due) + pattern watch — ~2–3 min on scan days, ~30 sec otherwise.

Typical total: **3–4 min** on non-scan days, **5–7 min** on scan days.

## Cost

| Call | Model | Purpose | Approx. cost |
|---|---|---|---|
| Main briefing | Sonnet + web search | Top stories, sectors, week ahead | ~$0.08 |
| Deep dive | Sonnet + web search | Long-form wildcard analysis | ~$0.05 |
| Signal scan | Sonnet + web search | Voices RSS + X (weekly) | ~$0.07/week |
| Pattern watch | Haiku | Cross-sector speculation | ~$0.001 |

**~$0.13/day** for daily calls, plus ~$0.07/week for Signal Scan = roughly **$3–4/month** running weekdays only.

Costs are billed to your own Anthropic account. Each user of this template is fully independent.

## Running locally

If you prefer to run on your own machine (e.g. served via Tailscale):

```bash
pip install anthropic pyyaml feedparser
export ANTHROPIC_API_KEY="sk-ant-..."
python3 morning_dashboard_v4.py
```

Output goes to `~/briefings/` by default. Override with:

```bash
export BRIEFING_OUTPUT_DIR="/path/to/folder"
```

Serve it locally:

```bash
python3 -m http.server 8080 --directory ~/briefings
```

Force regeneration of a day that already exists:

```bash
python3 morning_dashboard_v4.py --force
```
